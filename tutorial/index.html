PPV Stream — Page Guide

Explains the function of every page for both users and administrators. All mockup links open a static annotated replica of the live page.

Design notes: All pages share a unified Bootstrap 5.3.3 layout with a consistent navbar, light/dark toggle (data-bs-theme), and platform name injection via theme.js. User pages use a neutral navbar; admin pages use a red (bg-danger) navbar with an ADMIN badge.

User Pages

Browse Videos

/public/browse.html Open Mockup ↗

The main catalog. Users can search, filter, and sort all available videos, then click a card to open the Watch page.

UI ElementFunction
Navbar brand + linksConsistent navigation; active link highlighted
Search inputFilters by title, creator name, description (debounced 200 ms)
Sort dropdownNewest / Cheapest / Most expensive / A–Z
Video cardsClickable → goes to watch page; shows ▶ Ready or ⏳ Processing badge
Free badgeGreen text — video costs $0 (no payment needed)
Price displayIn Rupiah using live USD→IDR rate from /api/kurs
User barShows logged-in username; shows "Sign In" link if not logged in
Theme toggle (☀️/🌙)Switches light/dark; saved to localStorage

User Dashboard

/public/dashboard.html Open Mockup ↗

Central hub for content creators: manage profile, upload videos, configure pricing, and grant access.

SectionFunction
Profile & Payout InfoEmail (read-only), WhatsApp, bank account, EVM wallet, preferred chain, bio. Used for fiat/crypto disbursement.
Change PasswordRequires current password + new password confirmation. Uses POST /api/change_password.
My Videos & AccessLists uploaded videos with HLS status, inline edit form (title/price/description), and per-video allowlist.
Upload VideoMultipart form upload of MP4. Server transcodes to HLS automatically. Supports title, price, description.
Grant Access (Allowlist)Pick a video + enter username → grants access without payment. Uses POST /api/allow.

Wallet

/public/wallet.html Open Mockup ↗

Manage platform balance. Top-up via crypto or bank, withdraw to bank/crypto, transfer to another user.

Tab / ElementFunction
Balance cardShows available balance in USD and IDR equivalent. Refresh button reloads from server.
Deposit / Top Up tabChoose amount + method (x402 crypto / bank / QRIS). After submit, shows payment instructions.
Withdraw tabRequest withdrawal; destination is taken from Profile page. Admin processes within 1–2 business days.
Transfer tabSend balance to another user by username. Includes real-time username lookup.
History tabFull transaction log: date, type, amount, status, note. Loaded from /api/wallet/history.

Watch / Payment

/public/watch.html Open Mockup ↗

The pay-per-view page. Shows the HLS player if the user has access; otherwise shows a 3-tab payment panel.

ElementFunction
Locked player overlayShown before payment. Displays price in USD + IDR. ?ref= badge shown if referral link was used.
💳 Wallet tabInstant payment from wallet balance. Deducted immediately; access granted at once.
⛓ Crypto (x402) tabInitiates an x402 on-chain payment. Shows address + chain ID. Access confirmed automatically after tx.
🏦 Bank Transfer tabUser confirms after manual bank transfer. Access granted after seller verifies receipt.
HLS playerStreams via hls.js after access is confirmed. Falls back to native HLS for Safari.
Video descriptionShows creator's description below the player if available.

Admin Payments

/public/admin/payments.html Open Mockup ↗

Monitor all payment transactions. Approve or reject pending fiat payments and trigger seller disbursement.

ElementFunction
Search + filtersFilter by status (pending/completed/rejected) and method (x402/bank/QRIS/wallet).
Payments tableShows all transactions; pending rows have a Process button.
Process modalAdmin enters a note, then clicks Approve (credits seller) or Reject. Calls POST /api/admin/disburse.

Admin Settings

/public/admin/settings.html Open Mockup ↗

Configure platform branding, verify SMTP, set the exchange rate, and change the admin password.

SectionFunction
Platform Name & TaglineSaved to localStorage via theme.js. Applied to all pages via [data-platform] attributes. No DB migration needed.
SMTP SettingsRead-only display of server-side SMTP config (.env). Send Test Email button verifies current config.
Exchange RateSets USD → IDR rate used for price display sitewide. Stored server-side via POST /api/admin/kurs.
Change Admin PasswordRequires current password; calls same /api/change_password endpoint as users.

Admin Wallets

/public/admin/wallet.html Open Mockup ↗

Manage user wallet requests: approve withdrawals, view deposit history, and inspect user balances.

TabFunction
Withdrawal RequestsLists pending/completed/rejected withdrawal requests with user destination info. Process button opens modal.
Process modalComplete (manual transfer done) or Reject with optional note. Calls POST /api/admin/wallet_action.
Deposit HistoryAll deposit transactions across all users with invoice IDs and statuses.
User BalancesSearch all user balances, total deposited, and total withdrawn. Read-only overview.
Flow summary: User registers → browses videos → watches free or pays (wallet / x402 / bank) → video unlocks. Creators upload videos, set prices, and grant manual access. Affiliates share referral links and earn 10% per sale. Admin reviews fiat payments, approves withdrawals, configures platform settings.
